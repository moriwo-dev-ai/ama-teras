# コントリビューションガイド

AMA-teras へのコントリビューションを歓迎します。本体は AGPL-3.0([LICENSE](LICENSE))、
名称の扱いは [NOTICE.md](NOTICE.md) を参照してください。

## DCO(Developer Certificate of Origin)

PRの提出により、投稿コードが自分の作成物(または再配布可能なライセンス下で入手したもの)であり、
**AGPL-3.0 互換ライセンスでの提供に同意**したものとみなします。
コミットには `Signed-off-by: 名前 <メール>` を付けてください(`git commit -s`)。

## 聖域(保護領域)の方針 — 重要

このプロジェクトには**聖域**(承認機構・進化ガード・鍵素材の扱い・スコープ制御など、
「常に人間が止められる」性質を守るためのコード)があります。一覧と理由は
[docs/PROTECTED.md](docs/PROTECTED.md) を参照してください。

- **聖域を変更するPRは原則却下**します(安全機構の弱体化はレビューコストが非対称に高いため)。
  本当に必要な場合は、まず Issue で「なぜ聖域の変更なしに実現できないか」を説明してください
- 聖域は本体アプリ自身の自己進化からも、通常チャット経路の自動承認からも機械的に守られています。
  同じ規律を人間のPRにも適用します

## 歓迎するPR

- **コア改善**(バグ修正・性能・テスト・ドキュメント): 大歓迎です。
  ソース実行の開発者が生んだ改善を公式リリースで全員へ届けるのがこのプロジェクトの設計です
- **ツールプラグイン**: 本体へ直接ではなく、**コミュニティレジストリ**
  (amateras-registry・準備中)への投稿を推奨します(自己完結・環境非依存のため)。
  本体の組み込みプラグインに加えるべき汎用性があるものは Issue で相談してください
- **UIテーマ**: コードではなく宣言的データ(JSON)での共有を推奨します

## 開発の流れ

```bash
git clone <このリポジトリ>
npm install
npm run typecheck   # tsc(node / web / remote-ui の3構成)
npm test            # vitest(全テスト緑が必須)
npm run build       # 本体+スマホUI
```

- TypeScript strict・`any` 禁止(やむを得ない場合は理由コメント)
- エラーは握りつぶさない(ユーザーに見えるエラー表示をUIに用意)
- コメントは「なぜ」だけを書く
- 1 PR = 1 目的。機能単位でコミットし、動かない状態のコミットは避けてください

## レビュー基準(severity方式)

本体のレビュー文化は自己レビューゲートと同じ **severity 方式**です:

| severity | 意味 | PRへの影響 |
|---|---|---|
| **high** | 要件を満たさない・壊れている・安全性を損なう | 解消するまでマージしない |
| **medium** | 動くが品質上望ましくない | 指摘・修正推奨(マージは可) |
| **low** | 好みの問題 | 参考コメント |

- 新規コードには**テスト必須**(エージェントループ・プロバイダ層・進化マネージャ・
  ガード類は特に)。既存テストを壊す変更は理由の説明が必要です
- CI相当のローカル確認: `npm run typecheck` + `npx vitest run` の全緑をPR前に確認してください
