# M10 手動確認手順(スマホWebアクセス)

前提: M10 のコードがコミット済みで、**Windows側で一度 `npm run build` を実行済み**であること
(remote-ui の Vite ビルドはサンドボックス環境では実行できないため未実行。
`out/remote-ui/index.html` が存在しない場合、リモートUIは 404 になる)。

## A. PC単体での確認(iPhoneサイズのChrome)

1. `npm run build` → `out/remote-ui/index.html` が生成されていることを確認
2. `npm run dev` でアプリを起動
3. 設定 → 「リモートアクセス(スマホ・Tailscale経由)」
   - ポートが 8787 であることを確認 → **有効にする**
   - 「待受中 :8787」バッジになること
   - ペアリングトークン(64文字)が表示されること。**この画面にしか出ない**のでコピーしておく
4. Chrome で `http://localhost:8787/` を開く
   - DevTools (F12) → デバイスツールバー → iPhone 14 Pro 等を選択
   - トークン入力画面が出る → 3 のトークンを貼り付けて「接続」
   - ヘッダに「接続中」バッジが出ること
5. **認証の確認**
   - シークレットウィンドウで `http://localhost:8787/api/status` を直接開く → 401 JSON
   - `curl http://localhost:8787/api/status -H "Authorization: Bearer <トークン>"` → 200 JSON
   - でたらめなトークンで5回失敗 → 6回目は 429(60秒バン)
6. **チャット同期**
   - スマホ画面(Chrome)から「1+1は?」を送信
   - デスクトップのチャットにも同じ会話がストリーミングされること
   - 逆にデスクトップから送信 → スマホ側にも流れること
   - 実行中にスマホの「停止」でキャンセルできること
7. **承認の競合(first-response-wins)**
   - 自動承認を全てOFFにして、チャットで「hello.txt を作って」
   - デスクトップに承認ダイアログ、スマホの承認タブにもバッジ+カードが出ること
   - スマホ側で「許可」→ デスクトップのダイアログが**自動で閉じる**こと
   - 次の承認はデスクトップ側で応答 → スマホのカードが消えること
   - スマホの承認カードに diff / 警告 / systemスコープバナー(fullPc時)が出ること
8. **プランモード**: スマホでモードを「プラン」にして送信 → ツールが実行されず計画だけ返ること
9. **進化タブ**
   - スマホから「CSVをMarkdown表にする」等で進化ジョブを投入(またはチャット経由)
   - ジョブの状態遷移(queued→generating→verifying→awaiting_promotion)が流れること
   - 昇格承認カード(diff付き)がスマホの承認タブに出る → スマホで承認 →
     デスクトップの昇格ダイアログも閉じ、昇格完了まで進むこと
10. **監査タブ**: scopeMode を fullPc にし、プロジェクト外パスの操作を1回承認 →
    スマホの監査タブに hard-deny / 承認 / 実行結果のエントリが出ること(再読込ボタン)
11. **SSE再接続**: `npm run dev` を再起動(またはリモートを無効→有効)→
    スマホ側が「再接続中…」→ 自動で「接続中」に戻り、履歴が snapshot で復元されること
12. **トークン再生成**: 設定 → トークン再生成 → スマホをリロード →
    旧トークンでは 401(トークン入力画面)になり、新トークンで再接続できること
13. **無効化**: リモートを無効にする → `http://localhost:8787/` に接続できないこと(listenしない)

## B. iPhone実機(Tailscale経由)

1. PC と iPhone に Tailscale をインストールし、同一アカウントでログイン
2. PC の Tailscale 管理画面(またはトレイ)で MagicDNS 名を確認(例 `mypc.tailxxxx.ts.net`)
3. MyCodex 設定 → リモートアクセス有効化 → ホスト名欄に MagicDNS 名を入力 →
   接続URL(`http://mypc.tailxxxx.ts.net:8787/#t=<トークン>`)をコピー
4. URL を iPhone へ渡す(AirDrop / 自分宛メモ等。**QRコードは後続タスク**)
5. iPhone の Safari で開く → `#t=` からトークンが自動読込されて接続されること
6. 共有メニュー → 「ホーム画面に追加」 → スタンドアロン起動できること
   (トークンは localStorage 保存済みなので再入力不要)
7. A-6〜A-10 と同じシナリオをスマホ実機で一通り
8. **Wi-Fi を切って 4G/5G で再アクセス** → Tailscale 経由で届くこと
9. Tailscale を切った状態(社外ネットワーク等)からはそもそも到達できないこと
10. Windows ファイアウォールの注意: 初回 listen 時に許可ダイアログが出たら
    「プライベートネットワーク」のみ許可。tailnet 外からの到達は Tailscale が遮断する

## 確認済み(自動テスト)

- REST/SSE 全ルートのトークン認証(401/429)・allow-session の allow 降格
- 静的配信のパストラバーサル(`../`・`%2e%2e`・`..%5c`)で out/remote-ui の外に出られないこと
- SSE の snapshot → イベント中継、EventBus 購読解除
- トークン再生成で旧トークンが即失効すること
- `vitest run` 235件(M10時点)
