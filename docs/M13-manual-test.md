## 既知の制約(仕様)

- MCPはstdio起動型のみ(HTTP/SSEリモートサーバーはv2)
- MCPツールはパスベースのスコープ判定(M9)対象外。risk写像+サーバー単位信頼+承認で制御
- annotations の無いMCPツールは exec 扱い(自動承認していない限り毎回確認)
- 読み取り専用サブエージェント(dispatch_agent の read)からはMCPツールは使えない
  (承認を挟まない経路のため。work モードでは承認付きで使える)
- 記憶(memory)・計画(plan)はサブエージェントからは書けない(親が単独管理)
