# M18 手動確認(モデル自動切替: planner/worker/escalation)

自動テスト(vitest・typecheck 3構成)と実機ベンチ②(帯別トークン実測 →
docs/autonomy-comparison.md)は 2026-07-06 に実施済み。以下は体感確認用。

## 設定

- [ ] Settings →「モデル自動切替(役割ごとにモデルを使い分け)」をON →
      planner / worker / escalation の3帯とプリセット2種(高品質重視・コスパ重視)が出る
- [ ] 帯のプロバイダを選び直すとモデルが「既定」に戻る(誤モデル引き継ぎ防止)
- [ ] APIキー未登録のプロバイダを帯に選ぶと ⚠ 警告が出る
- [ ] 下部に「現在: メイン会話=… 実行サブ=… 格上げ先=…」の一覧が出る
- [ ] OFFに戻すと従来の単一モデル動作(既定はOFF。何も変わらないのが正)

## 動作

- [ ] ON状態でチャットすると、アシスタントの吹き出しに小さな「planner」ラベルが付く
- [ ] 「実装は dispatch_agent(work)へ委譲して」を含む依頼で、右ペイン「エージェント」タブに
      worker のサブエージェントが走る(main ログの [usage] で session=subagent-N が worker 帯)
- [ ] ONにした直後の送信で「ℹ️ モデル切替を検知」カードが出ることがある(planner帯への切替。
      M16-1 の正常動作)
- [ ] worker帯のキーだけ未登録にすると「⚠ … APIキーが未設定 — サブエージェントは planner 帯で
      代行します」のカードが出て、作業自体は止まらない

## エスカレーション(再現しづらい)

- [ ] worker が エラー終了 / ターン上限 / ツール失敗3連続 になると
      「⤴ サブエージェント#N の実行が難航…上位モデルへ格上げして再試行します」カードが出る
- [ ] エージェントタブの該当行に「⤴ 格上げ」バッジが付く
- [ ] `%APPDATA%\amateras\audit.jsonl` に `"tool":"model-escalation"` が記録される
- [ ] 格上げは1タスクにつき最大回数(既定1)まで。格上げ先でも失敗したら普通にエラーで止まる

## 併存・非波及

- [ ] M16フォールバック(残高切れ)はON時も各帯で効く(1会話1回の制限は共通)
- [ ] 進化ジョブ(request_capability)は modelPolicy に関係なく本体設定のモデルで動く
- [ ] OFFのユーザーは今までと完全に同じ(既存テスト不変で担保)
