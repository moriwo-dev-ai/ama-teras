# M17 手動確認(リブランド AMA-teras・自律モード・左ペインソート)

自動テスト(vitest 515件・typecheck 3構成)と、Claude Code による実機CDP検証
(userData移行後のキー復号・セッション19件引き継ぎ、自律モードのモーダル/警告バー/ON-OFF、
左ペインの更新降順)は 2026-07-06 に実施済み。以下は体感確認用。

## M17-1: リネーム(MyCodex → AMA-teras)

- [ ] 起動するとウィンドウタイトル・ヘッダが「AMA-teras」になっている
- [ ] **APIキーがそのまま使える**(設定 → APIキーが「登録済み」表示。旧 `%APPDATA%\mycodex` から
      `%APPDATA%\amateras` へ初回起動時に自動移行される。鍵素材の Local State も移行済み)
- [ ] 過去のセッション一覧・履歴が全部残っている
- [ ] スマホ(remote-ui)は再ログイン不要でそのまま繋がる(トークン保存キーは不変)。
      表示名・ホーム画面アイコン名が「AMA-teras」になっている(PWAは再追加が必要な場合あり)
- [ ] エージェントに「計画を立てて」と頼むと `AMATERAS_PLAN.md` が作られる。
      旧 `MYCODEX_PLAN.md` / `MYCODEX.md` が残っているプロジェクトでは内容が読み込まれ、
      次の書き込みで新名へ自動リネームされる
- [ ] 旧 `%APPDATA%\mycodex` は消えずに残っている(ロールバック用。不要になったら手動削除可)

## M17-2: 自律モード

- [ ] 送信欄の左に「🔒 通常」ボタンがあり、クリックで警告モーダルが出る
- [ ] モーダルの「リスクを理解しました」にチェックするまで「有効化(自己責任)」が押せない
- [ ] 有効化するとチャット上部にオレンジの警告バーが常時表示され、ボタンが「🔓 自律モード」になる
- [ ] ON中は write/exec のツール実行で承認ダイアログが出ない(勝手に進む)
- [ ] fullPc スコープの system 操作(ワークスペース外の読み書き)も承認なしで進む
- [ ] **拒否されることの確認**(ON中でも):
  - 「%APPDATA%\amateras\secrets.json を読んで」→ 拒否される
  - 「C:\Windows\win.ini を書き換えて」→ 拒否される(既存システムファイルの上書き)
  - 「format c: を実行して」→ 拒否される(破壊的コマンドdenylist)
  - 「C:\Windows\Temp に新しいメモファイルを作って」→ これは通る(新規作成は許可)
- [ ] `%APPDATA%\amateras\audit.jsonl` にON中の全操作が `"autonomous":true` で記録されている
- [ ] バーの「OFFにする」でワンクリック解除
- [ ] アプリを再起動すると必ずOFFに戻っている。セッション切替・新規チャットでもOFFに戻る
- [ ] スマホ: ヘッダの🔒をタップ → 同等の確認(チェック必須)→ ON。デスクトップとバーが同期する
- [ ] 進化ジョブ(request_capability)は自律モードONでも従来どおりの承認フロー

## M17-3: 左ペインのソート

- [ ] プロジェクトの並びは「最後に更新したものが上」(選択しただけでは並びが変わらない)
- [ ] 下の方のプロジェクトのセッションを開いても、そのプロジェクトは上に飛ばない
      (プロジェクト名が青字+「現在」バッジになるだけ)
- [ ] そのセッションでメッセージを送る(=更新される)と、次の一覧更新でそのプロジェクトが先頭へ来る

## 回帰確認(ついで)

      名前を変えたければショートカットをリネーム)
