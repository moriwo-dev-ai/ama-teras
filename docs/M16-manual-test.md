## M16-1: モデル切替時の自動compaction

- [ ] 長めの会話(数往復・ツール実行あり)の途中で Settings からプロバイダ or モデルを変更 →

## M16-2: 自動リトライ+プロバイダフォールバック

- [ ] (再現しづらい)API混雑時(529等)に「ℹ️ 一時的なAPIエラーのため再試行します(n/3、Xs待機)」
      が出て、成功すれば何事もなく続行される
- [ ] リトライ待機中に「停止」を押すと即キャンセルされる
- [ ] Settings →「フォールバック(残高切れ時の自動切替)」ON → 切替先プロバイダ/モデルを設定。
      切替先のAPIキーが登録済みであること
- [ ] 残高切れ発生時(またはキー残高を使い切ったタイミング)に
      「⚠ 残高/課金エラーを検知したため、フォールバック(…)へ切り替えて続行します」が出て
      作業が止まらない
- [ ] `%APPDATA%\mycodex\audit.jsonl` に `"tool":"provider-fallback"` の行が記録されている
- [ ] 同じ会話で2回目の残高エラーはフォールバックせず通常のエラー停止(往復ループ防止)
- [ ] フォールバック後も Settings の本体プロバイダ設定は変わっていない

## M16-3: markdown表示+コードブロックのコピー

- [ ] 「フィボナッチをTypeScriptで書いて」→ 返答のコードが枠付きブロックで表示され、
