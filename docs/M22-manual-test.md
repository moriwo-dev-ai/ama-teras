# M22 手動確認(複数プロジェクト同時実行)

自動テスト(レース系5件含む624件・typecheck 3構成)は2026-07-06夜間に全合格。
実機ではイベント帰属・会話分離・切替往復・エラー表示経路を確認済み
(※Anthropic残高切れのため、実LLM応答での複数同時ランの目視確認は残)。

## 並行実行の基本

- [ ] プロジェクトAで長いタスクを開始 → 実行中に左ペインでプロジェクトBのセッションを開ける
      (Aは止まらない。左ペインのAに◌スピナーが回る)
- [ ] Bで別のタスクを開始 → 2つ同時に走る(左ペインに◌が2つ)
- [ ] Bのチャットには Bのイベントだけが流れる(Aのツールカード・応答が混ざらない)
- [ ] 実行中のAへ戻ると、実行状態(停止ボタン・経過時間・追加指示)に復帰する
- [ ] Aを停止してもBは走り続ける(バックグラウンドプロセスも会話単位)

## 承認の帰属

- [ ] 2会話から同時に承認要求が来ても、ダイアログに「出所: 📁プロジェクト名 — 会話タイトル」が
      表示され、どちらの承認か分かる(スマホの承認カードも同様)
- [ ] それぞれ承認/拒否しても取り違えが起きない

## 会話単位の状態

- [ ] 会話Aで自律モードON→実行中にBへ切替 → Bは通常(OFF)。Aへ戻るとONのまま
- [ ] フォールバック(M16)は会話ごとに1回まで(片方で発動しても他方は影響なし)
- [ ] 実行中セッションの削除は「実行中のセッションは削除できない」と拒否される
- [ ] 実行中のworkspaceでのチェックポイント復元は拒否される(完了後は可)

## しきい値と負荷

- [ ] 同時6ラン目で「⚠ 同時実行が6件…」の注意カード(実行自体は止まらない)
- [ ] 実行中に視聴プロジェクトを切り替えても(config.workspaceが変わっても)、
      実行中ランのファイル操作は元のプロジェクトに閉じたまま(workspace束縛)

## 後方互換

- [ ] 1会話だけで使う分には従来と同じ操作感(送信→応答→ツールカード→停止)
