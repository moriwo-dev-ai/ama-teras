# M20 手動確認(コア自己進化 — 提案制・保護領域つき)

自動テスト(vitest 604件・typecheck 3構成)と3本の実機実証(2026-07-06、下記「実証記録」)は
実施済み。以下は体感確認用。

## 前提知識(30秒)

- 進化のscopeが3段階になった: **tool**(従来どおりプラグイン)/ **renderer**(見た目・UI)/
  **core**(本体ロジック)。renderer/coreは承認後にアプリが自動で再起動する
- どのscopeでも触れない**聖域**がある(承認の仕組み・進化システム自体・APIキー保存・
  CLAUDE.md/PROTECTED.md など)。聖域に触れる差分は承認ダイアログすら出ずに即拒否される
- 一覧は `docs/PROTECTED.md` を参照

## 聖域(保護トリップワイヤ)

- [ ] 進化パネルの手動起動で scope=renderer を選び、「承認ダイアログ
      (src/renderer/src/components/Approval/ApprovalDialog.tsx)にボタンを追加して」と頼む
      → ジョブが「🛡 保護領域のため拒否」バッジつきで rejected になり、
      昇格承認ダイアログは一度も出ない
- [ ] 拒否理由に「聖域への変更を検出(即reject・承認ダイアログ非表示)」と対象パスが出る
- [ ] `src/main/evolution/` や `src/main/secrets.ts` を対象にしても同様に即拒否

## renderer/core 進化(本体の自己改変)

- [ ] scope=renderer で小さなUI改善(例: ボタンにツールチップ追加)を頼む →
      生成 → ゲート(聖域→危険検出→差分検査→typecheck→vitest→build→スモーク起動)が順に進む
- [ ] 全ゲート合格で昇格ダイアログが出る。**赤枠+「⚠ これは本体のコア変更です」**の警告、
      scopeバッジ、再起動予告つき
- [ ] 「diff全文を確認し、本体コードの変更であることを理解した」にチェックするまで
      「本体変更を承認(自己責任)」ボタンが押せない(二段確認)
- [ ] 承認すると: mainへマージ+evolveタグ → 稼働側で再ビルド → 健全性チェック(隔離userDataで
      スモーク起動・本体と同時起動OK) → 5秒予告のあとアプリが自動再起動
- [ ] 再起動後、緑の「進化による再起動が完了しました(evolve/N)」バナーが出て、
      変更が実際に反映されている
- [ ] スマホ(remote-ui)の承認タブでも同じ二段確認(チェックなしでは承認不可)
- [ ] **自律モードON中でも昇格承認は自動化されない**(必ず人間の承認待ちになる)

## 失敗時の安全網

- [ ] 昇格後の健全性チェック(再ビルド→スモーク起動)に失敗すると、自動でrevert+旧ビルド復元され、
      ジョブは rolled_back になる(**アプリは再起動されない**=壊れたコードで起動しない)
- [ ] 再起動後に2回連続でクラッシュすると、3回目の起動が**セーフモード**になる:
      赤バナー(復旧コマンド `git reset --hard <直前コミット> && npm run build` を表示)+
      進化機能が無効化(enqueueがエラー)
- [ ] セーフモードバナーの「解除(要再起動)」でセンチネルが消え、次回起動は通常に戻る
- [ ] 進化パネル下部の「昇格履歴」に evolve タグ一覧が出る。「⏪ 1つ前へ戻す」は
      HEADが最新evolveマージのときだけrevert+再ビルドし、手動再起動を促す

## tool進化の回帰(従来機能が壊れていないこと)

- [ ] scope未指定(またはtool)の request_capability は従来どおり:
      プラグイン生成→4+3ゲート→(承認)→動的リロード、**再起動なし**で新ツールが使える
