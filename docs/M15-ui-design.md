# M15: UIリニューアル — 3ペインレイアウト(Codex/Claude Codeデスクトップ風)

## 目的

ユーザー提供のスクリーンショット(Codexデスクトップ・Claudeデスクトップ)を参考に、
現在の1カラム+ヘッダボタン群のUIを3ペインに再構成する。**機能追加ではなくUIの再配置**
— main側のロジック・IPCは原則変更しない(追加のみ)。

```
┌────────────┬──────────────────────┬────────────────┐
│ 左: ナビ    │ 中央: チャット         │ 右: コンテキスト │
│ ・新規チャット│ (既存のチャットUI)     │ ・ファイルプレビュー│
│ ・検索      │                      │ ・環境ウィジェット │
│ ・プロジェクト│                      │ ・既存パネル群を  │
│   └セッション│                      │   タブで集約     │
└────────────┴──────────────────────┴────────────────┘
```

## M15-1: 3ペインの骨格

- 左右ペインはドラッグでリサイズ可・ボタンで折りたたみ可(状態は localStorage)
- ウィンドウ幅が狭い時は左右ペインをオーバーレイ化(remote-uiは対象外・1カラムのまま)
- 既存機能はすべて到達可能なまま(この段階では配置換えだけ。既存E2Eが通ることが完了条件)

## M15-2: 左ペイン — プロジェクトとセッション

- **プロジェクト = workspace ディレクトリ**。セッション(M12-1)を workspace ごとにグルーピングして
  ツリー表示(プロジェクト名はフォルダ名、セッションはタイトル+相対時刻)
- クリックでセッション切替。**workspace が異なるセッションを開いたら workspace も自動で切替**
  (M12の既知の制約「復元してもworkspaceは切り替わらない」をここで解消。
  エージェント実行中は切替不可の既存ガードは維持)
- 「新規チャット」: 現在のプロジェクトで新規セッション。プロジェクト横の「+」で
  フォルダ選択→そのworkspaceで新規セッション
- 検索ボックス: セッションタイトル+履歴本文の単純部分一致(main側に sessions:search IPC を追加。
  インデックス構築はしない — ファイル数百想定の逐次検索で十分)
- セッションの右クリック: 名前変更 / 削除

## M15-3: 右ペイン — ファイルプレビュー

- チャット内のファイル参照をクリック可能に:
  - ツールカード(write_file/edit_file/read_file等)のパス
  - メッセージ本文中のパスらしき文字列(workspace配下に実在するもののみリンク化)
- クリック → 右ペインにプレビュー:
  - `.md`: レンダリング表示(`react-markdown` + `remark-gfm` を依存追加)
  - コード/テキスト: 等幅・簡易ハイライト(依存を増やさず既存のdiff表示スタイルを流用)
  - 画像(M14成果物): そのまま表示
- ヘッダ: ファイル名 / 再読込 / 「フォルダで表示」(`shell.showItemInFolder`)/ 閉じる
- 読み取りは新IPC `file:preview`(workspace配下+実在チェック。workspace外は
  fullPc時のみ・M9のスコープ判定を通す)

## M15-4: 右ペイン — 環境ウィジェットとパネル集約

- 上部に環境ウィジェット: workspaceパス / gitブランチ / 未コミット変更数 /
  直近チェックポイント数 / 計画進捗(n/m)。gitはmainの軽量IPC(`workspace:gitStatus`)で取得
  (git無しworkspaceでは非表示)
- 現在ヘッダボタンで開く**計画 / エージェント / 進化 / Debug / 監査系パネルを右ペインのタブへ集約**
  (ファイルプレビューもタブの1つ。承認ダイアログはモーダルのまま変更しない)
- 進行中イベント(承認待ち・進化ジョブ・サブエージェント)はタブにバッジ表示

## M15-5: 仕上げ

- キーボード: Ctrl+B(左)/ Ctrl+J(右)でペイン開閉、Ctrl+K で検索フォーカス
- docs/M15-manual-test.md(リサイズ・折りたたみ・セッション切替でのworkspace連動・
  mdプレビュー・環境ウィジェット・既存パネルの到達性)
- USER-GUIDE.md の画面説明を新レイアウトに更新(スクショ差し替えはユーザーと後日)
- PROGRESS.md 更新

## 制約・非対象

- renderer中心の変更。main側は新IPC(sessions:search / file:preview / workspace:gitStatus)の
  追加のみで、既存IPCの変更禁止
- remote-ui(スマホ)は現行1カラムのまま
- テーマ・配色の刷新はしない(既存Tailwindトークンの範囲で)
- エディタ機能(ファイル編集)は持たせない(プレビューのみ。編集はAIに頼むか外部エディタ)

## 受け入れ基準

- 既存全テスト+新規テスト合格 / typecheck 3構成
- 既存機能すべてに新レイアウトから到達できる(既存CDP E2Eの改修込みで全通過)
- セッション切替で workspace が追従する
- チャット内の .md クリック → 右ペインでレンダリング表示される
