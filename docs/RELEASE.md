# リリース手順

**`npm run release` 一本。手で順番を思い出さない。**

```bash
# 1. リリースノート本文を書く(Markdown)
#    何を足したか・何を直したか。実害の出た不具合は隠さず書く

# 2. 下書きまで作る(既定)
npm run release -- --minor --notes-file notes.md --title "AMA-teras v1.1.0 — 副題"

# 3. GitHub で内容を確認して「Publish release」を押す
#    (--publish を付ければ公開まで自動。押した瞬間に全利用者へ更新通知が飛ぶ)
```

`--patch`(バグ修正のみ)/ `--minor`(機能追加)/ `--major`(互換性を壊す)/ `--version v1.2.3`。
`--dry-run` で何をするかだけ見られる。

## スクリプトがやること(そして、なぜその順番か)

| 手順 | 中身 | なぜ |
|---|---|---|
| 0 | gh の存在・作業ツリーが綺麗・main ブランチ・ノート本文の指定を確認 | 途中で落ちると中途半端な状態が残る。**前提は最初に潰す** |
| 1 | 公開済み最新リリースから次の版を決める。同じタグがあれば中止 | 前回の版を人間が覚えない。既存リリースの上書き事故を防ぐ |
| 2 | `package.json` を上げてコミット・push | **ビルドより先**。逆にすると中身が古い版のインストーラができる |
| 3 | typecheck + テスト | 壊れたものを配らない |
| 4 | `npm run dist` でインストーラをビルド | |
| 5 | 下書きリリースを作り、インストーラを添付 | |
| 6 | 添付されたことを `gh` で**実際に確認** | 思い込みで終わらせない(下の事故を参照) |
| 7 | `--publish` のときだけ公開 | 公開 = 全利用者への更新通知。人間の判断を最後に置く |

## なぜスクリプトにしたか(実際に起きた事故)

v1.1.0 のリリース作業で、**インストーラの添付を忘れた**。更新バナーは出るのに、行った先に
落とすものが無い——という最悪の壊れ方をする。さらに README のダウンロードリンクが v1.0.0 の
実ファイルに固定されていて、新版を出しても古い版が落ちてくる状態だった(修正済み。
README は `/releases/latest` を指す)。

手順が多く、順番も効く作業を人間の記憶に置くと必ず落ちる。だから機械に固定した。

## 更新は利用者にどう届くか

- アプリは `releases/latest` のタグと自分の `package.json` の version を比べ、新しければ
  **上部にバナーを出すだけ**(自動ダウンロード・自動更新はしない)
- **下書きリリースは誰にも通知されない**(`releases/latest` は下書き・プレリリースを返さない)。
  通知が飛ぶのは「Publish」を押した瞬間
- インストーラ版の利用者: バナー → リリースノート → `.exe` を上書きインストール
  (設定・APIキー・会話履歴・進化させたツールは userData 側なので引き継がれる)
- ソース版の利用者: `git pull && npm install && npm run build`

## 誰が実行できるか(公式リポジトリへの誤爆を防ぐ)

- **リリース先は `git remote origin` から決める**。公式リポジトリはハードコードしない。
  フォーク/クローンで実行しても、リリースは**自分のリポジトリ**に作られる
- `--repo` で上書きできるが、origin と違う先には `--repo-force` を明示しない限り進まない
  (フォーク元へリリースしようとする事故の防止)
- そもそも他人のリポジトリには書き込み権限が無いので `gh` が失敗する。上の2つは
  「権限で弾かれる前に、そこへ向かわない」ための歯止め
- **インストーラで入れた利用者はこのスクリプトを持っていない**
  (配布物に入るのは `out/` と `package.json` と `plugins/` だけ)

## 運営タブ(神々)からはリリースを完結できない

運営タブの「🐙 GitHub Releaseへ」は **リリースノートの下書きを作る/更新する**ところまで。
インストーラのビルドと添付はできない(神々にビルドと配布物のアップロードは持たせていない)。
配布まで含めたリリースは、必ずこのスクリプトで行うこと。
