# AMA-teras 使い方ガイド(初心者向け)

AMA-terasは、チャットで頼むとAIが**あなたのPCの中で実際に手を動かしてくれる**デスクトップアプリです。
ファイルを読み書きし、コマンドを実行し、アプリを作り、テストまで通します。
そして足りない機能があると、**自分で自分に新しい機能を作って組み込みます**(自己進化)。

---

## できること一覧

| カテゴリ | できること |
|---|---|
| コーディング | アプリ・スクリプト・テストの作成、バグ修正、リファクタリング(小〜中規模なら計画→実装→デバッグ→テストまで無介入で完遂) |
| ファイル操作 | 読む・書く・編集・検索(grep)・フォルダ整理。PC全体モードならプロジェクト外も(毎回あなたの承認つき) |
| コマンド実行 | npm・git など任意のシェルコマンド。devサーバのような常駐プロセスはバックグラウンド実行 |
| 計画的な長作業 | 計画ファイル(AMATERAS_PLAN.md)を自分で作って進捗管理。数十ターンの作業も迷子にならない |
| 並列作業 | サブエージェント最大3体に別々の作業を同時にさせる |
| 中断・再開 | アプリを閉じても(落ちても)会話が残る。再起動→セッション選択→「続けて」 |
| 巻き戻し | ファイル変更のたびに自動チェックポイント。失敗したらワンクリックで復元 |
| 自己進化 | 「◯◯できるツールが欲しい」→ 自分でツールを作り、テストし、あなたの承認を得て組み込む |
| スマホ操作 | iPhoneから進捗確認・指示・承認(Tailscale経由、外出先からもOK) |
| モデル選択 | Claude(既定: claude-fable-5)と OpenAI を設定で切替 |
| 無料で始める | カード登録なし。Gemini / Groq / OpenRouter の無料枠APIで即開始(M27) |
| プラグイン共有 | 作ったツールをフォルダとして書き出し/他の人の共有ツールを検証ゲート付きで導入(M27) |

## 最初のセットアップ(5分・初回だけ)

1. **起動**: `AMA-teras Setup 0.1.0.exe` でインストールした場合はスタートメニューから。開発版は `npm run dev`
2. **APIキー登録**: Settings(⚙)→ APIキー欄に Anthropic(または OpenAI)のキーを貼って保存。
   キーはWindowsの暗号化機能で保存され、平文では残りません
3. **作業ディレクトリ**: Settings → 作業ディレクトリ で、AIに触らせたいフォルダを選ぶ。
   **ここが「AIの担当範囲」になります**(この外は原則触らない)
4. 以上。チャットに何か頼んでみてください:
   > 「このフォルダに何があるか教えて」

### お金をかけずに試す(無料APIモード・M27)

カード登録なしで始められます。Settings → 基本タブの緑色の「**無料で始める**」枠から:

1. プリセットを選ぶ(**Google Gemini が推奨**=無料枠が最も太い。ほかに Groq / OpenRouter)
2. 「APIキー取得ページを開く」→ 画面の3ステップ案内どおりにキーを作ってコピー
3. キーを貼り付けて保存 → 「**接続テスト**」ボタンで疎通確認(成否がその場で出ます)

プリセットを選ぶと**無料モード**が自動でONになり、無料枠を溶かしやすい機能が軽量化されます
(最大ターン数の既定15・品質レビューゲートOFF・モデル自動切替OFF・新ツールの自己生成OFF。
それぞれ設定で戻せます)。無料枠の上限に達すると「明日リセットされます」のような
平易なメッセージで止まります。

> 💡 M29: Gemini の既定モデルは現行世代 **gemini-3.5-flash** です(候補に 3.1 Flash-Lite /
> 2.5 Flash もあり)。接続テストが失敗したときは、実際に叩いたURL・HTTPステータス・
> レスポンス内容が表示されるので、その文言をそのまま添えて相談すれば原因がすぐ分かります。

> ⚠ 無料枠APIは入力データが提供元の学習に使われる場合があります。
> 機密情報を扱う作業には Anthropic / OpenAI の有料APIを使ってください(画面にも常時表示)。

## 画面の見方(M15: 3ペイン)

```
┌──────────┬──────────────────┬──────────────┐
│ 左: ナビ   │ 中央: チャット      │ 右: コンテキスト │
│ 新規チャット│ (会話・ツールカード) │ 環境ウィジェット  │
│ 検索(Ctrl+K)│                  │ 📄/計画/エージェント│
│ プロジェクト │                  │ /進化/Debug タブ │
│  └セッション│                  │               │
└──────────┴──────────────────┴──────────────┘
```

- **左ペイン**: プロジェクト(フォルダ)ごとの会話一覧。クリックで切替(作業フォルダも自動で追従)。
  右クリックで名前変更・削除。Ctrl+B で開閉
- **右ペイン**: 上に環境ウィジェット(フォルダ・gitブランチ・未コミット数・計画進捗)、
  下にタブ(ファイルプレビュー📄・計画・エージェント・進化・Debug)。Ctrl+J で開閉
- **ファイルプレビュー**: チャット内のファイルパス(ツールカードの📄や本文中の点線リンク)を
  クリックすると右ペインに中身が出る(.mdは整形表示・画像はそのまま)
- ペインの境界はドラッグでリサイズ可。ウィンドウが狭いときはオーバーレイになります

## 基本の使い方

### チャットで頼む

普通に日本語で頼むだけです。AIは必要に応じてツール(ファイル読み書き・コマンド実行)を使い、
その様子がチャット内にカードで表示されます。**「停止」ボタンでいつでも中断できます。**

AIの返答は整形表示(markdown)され、コードには言語ラベル付きの枠と **📋 コピー** ボタンが
付きます。押すと「✓ コピーしました」に変わり、コードだけがクリップボードに入ります
(スマホ版にも同じボタンがあります)。

### 承認ダイアログ — 一番大事な画面

AIが「何かを変える」操作をする前に、確認ダイアログが出ます:

- **色 = 危険度**: 読み取り(safe)/ファイル変更(write)/コマンド実行(exec)
- **diff表示**: ファイルを書き換える場合、何がどう変わるか差分で見える
- 選択肢は3つ: **許可**(1回だけ)/**セッション中許可**(この会話中は同じツールを聞かない)/**拒否**
- 面倒になったら Settings → 自動承認 で危険度ごとにOFFにできます
  (慣れるまでは write と exec は手動承認がおすすめ)

### 計画モード

送信欄の「plan」に切り替えると、AIは**実行せずに計画だけ**提示します。
内容が良ければ承認 → 通常モードで実行。大きめの依頼はまずplanで見るのが安全です。

### 自律モード(🔓) — 承認なしの全自動(M17・自己責任)

送信欄の左の「🔒 通常」を押すと**自律モード**にできます。ON中は承認ダイアログが一切出ず、
ファイル変更・コマンド実行・PC全体(fullPc時)の操作をAIが**確認なしで連続実行**します。

- ONにする時は警告モーダルで「リスクを理解しました」のチェックが必須です
- ON中はチャット上部に**オレンジの警告バー**が常時出ます(「OFFにする」でワンクリック解除)
- それでも**絶対に自動実行されないもの**: APIキー(secrets)の読み書き、アプリ設定領域の書き込み、
  Windowsシステム領域の既存ファイル上書き、`format`/`diskpart` などOSを壊す破壊的コマンド
- ON中の**全操作が監査ログ(audit.jsonl)に記録**され、スマホの「監査」タブからも見えます
- 状態は**その場限り**: アプリ再起動・セッション切替・新規チャットで必ずOFFに戻ります
- スマホからもヘッダの🔒でON/OFFできます(ONは同じ確認あり)

「自動承認を全部ON」との違い: 自動承認は fullPc の system 操作だけは毎回承認が残りますが、
自律モードはそれも自動化します。夜間の長時間自走で承認待ちで止まらないための最終手段です。

## 長い作業を任せる(ここがAMA-terasの本領)

### 実行中にできること(M21)

- **追加指示**: 実行中も入力欄は生きています。「あ、それも一緒にやって」と送ると
  「↩追加指示」として次のターンで反映されます(止める必要なし)。複数回OK
- **今なにしてる?が見える**: 指示の直下に「現在の状況」(スピナー+経過時間+
  最新の思考テキスト)がライブ表示。bash実行中は出力の末尾も流れ、30秒無音だと
  「⏳無応答」の注意が出ます(勝手にkillはしません)
- **止める**: 停止ボタンは従来どおり即キャンセル

### 複数プロジェクトを同時に走らせる(M22)

- 実行中でも左ペインで**別プロジェクトのセッションを開いて、別のタスクを開始**できます。
  実行中のセッションには◌スピナーが付き、クリックすればいつでも実行中の画面に戻れます
- チャット・承認・進捗はプロジェクト(会話)ごとに分離。承認ダイアログには
  「出所: 📁プロジェクト名」が出るので取り違えません
- 同時数に上限はありませんが、5個を超えると注意カードが出ます(APIレートと料金に注意)
- 自律モード(🔓)は会話ごと。Aで自律実行中にBへ移っても、Aは自律のまま走り続けます

### おまかせ設定(寝る前・外出前)

1. Settings → 自動承認 を全部ON
2. Settings → 最大ターン数 を 100 に(既定30。1〜200)
3. Settings → 編集後フック に `npm run -s test --if-present` を設定
   → ファイルを編集するたびにテストが自動実行され、失敗をAIが自分で見て直します
4. 頼み方のコツ: **「計画を立てて進めて」を添える** → AMATERAS_PLAN.md ができ、
   ヘッダ「計画」ボタンのパネルで進捗(n/m完了)が見えます
5. 承認待ちすら無くしたい場合は**自律モード(🔓)**をON(上記参照。リスク了承の上で)

### 止まりにくくする(自動リトライとフォールバック)

- API側の一時的なエラー(混雑・レート制限・タイムアウト)は**自動で最大3回まで再試行**します。
  チャットに「ℹ️ 一時的なAPIエラーのため再試行します」と出るだけで、何もしなくてOKです
- **残高切れ対策**: Settings → **フォールバック(残高切れ時の自動切替)** をONにして、
  切替先(例: OpenAI / gpt-5.1)を選んでおくと、残高・課金エラーを検知した瞬間に
  **その会話だけ切替先で自動続行**します(⚠カードで通知・audit.jsonlに記録)
  - 本体のプロバイダ設定は変わりません。発動は1会話につき1回。切替先のAPIキー登録が必要です
  - 一晩の放置作業でクレジットを使い切っても止まらないための保険です
- **セーフガード拒否の自動復帰(M26)**: Fable 5 は安全対策で応答を拒否(refusal)することが
  稀にあります。従来はそこでタスクが止まりましたが、いまは**同一プロバイダの1段下モデル
  (例: Fable 5 → Opus 4.8)で同じターンを自動でやり直します**(1会話2回まで・フォールバック
  設定なしでも発動・⚠カードと監査ログに記録)

### モデル自動切替 — 役割ごとに使い分けてコスパUP(M18)

Settings →「モデル自動切替」をONにすると、1つの依頼の中でモデルを役割分担させます:

| 帯 | 役割 | 推奨 |
|---|---|---|
| planner | 計画・重要レビュー・最終報告(メイン会話) | Fable 5 |
| worker | 実行の手数(ファイル編集・テスト回し=サブエージェント) | Sonnet 5 / Haiku 4.5 |
| explorer | 調査・ファイル探索(mode:"read"のサブエージェント。M26) | Haiku 4.5 |
| reviewer | 日常の品質レビューの監査役(M26) | Sonnet 5 / Haiku 4.5 |
| midEscalation | 中間の格上げ先(レビュー差し戻し3回目。M26) | Opus 4.8 |
| escalation | 最終の格上げ先 | Fable 5 |

- プリセット「高品質重視」「コスパ重視」からワンタップで設定できます
- worker が エラー/ターン上限/失敗3連続 になると、そのタスクだけ自動で上位モデルに
  格上げして再試行します(チャットに ⤴ カード・エージェントタブにバッジ・監査ログに記録)
- explorer / reviewer / midEscalation は未指定でも動きます(それぞれ worker / planner /
  escalation で代行)。設定 →「接続」タブの使用量に**帯別の累積トークン・概算コスト**が
  出るので、効果を見ながら調整できます(M26)
- 帯ごとにプロバイダ横断もOK(例: planner=Anthropic、worker=OpenAI)。その帯のAPIキーが
  必要です(未登録は設定画面に⚠、実行時は警告カードを出して planner が代行)
- OFF(既定)なら従来どおり単一モデルで、何も変わりません。進化ジョブは常に本体設定のモデルです

### 品質レビュー — 「動く」だけでなく「質」を守る(M19)

Settings →「品質レビュー」をONにすると、マイルストーン(計画項目の完了)ごとに
高モデル(planner帯)が**辛口の第三者レビュアー**として成果物を採点します:

- 4軸(コード品質 / 見た目UX / 要件達成 / テストの質)を1〜5で採点。UIが無いタスクでは
  UX軸は自動スキップ。Webアプリならscreenshotで実際の画面も確認します
- **合格判定は severity 方式が既定(M26)**: 指摘に high(要件を満たさない/壊れている)/
  medium(望ましくないが動く)/ low(好み)の深刻度が付き、**highがゼロなら合格**。
  差し戻されるのは high のみで、medium/low はカードに記録されるだけ。スコア平均は参考表示。
  従来の「平均閾値(既定4.0)方式」にも設定で戻せます
- 上限まで直らなければ残課題を提示して先へ進みます(自律モードOFFなら承認を求めます)
- 結果はチャットの🧪レビューカード(緑=合格/琥珀=差し戻し/赤=残課題。指摘ごとに
  severityバッジ)と監査ログで確認できます
- プリセット: しっかり(4.5/3周)・標準(4.0/2周)・軽め(3.5/1周)
- 差し戻しの修正は 1-2回目=worker、3回目=midEscalation、4回目以降=escalation と
  段階的に強いモデルへ引き継がれます(M26)
- コスト: 日常レビューは reviewer 帯(安価に設定可)が担当し、最終マイルストーン完了時・
  本体コア領域に触れる変更・完成時レビューだけ planner 帯で実施します(M26)。
  計画を使わない短いタスクは完了時に1回だけ

### 失敗したら巻き戻す

ファイル変更・コマンド実行のたびに自動でチェックポイントが取られています(gitの履歴は汚しません)。
Debugパネル → チェックポイント一覧 → 「復元」で、その時点のファイル状態に戻せます。
復元前の状態も自動退避されるので、行ったり来たりできます。

### 中断・再開

- アプリを閉じても、強制終了しても、会話はセッションとして自動保存されています
- 再起動 → チャット上部のドロップダウンからセッションを選ぶ → 「続きをやって」
- 「新規セッション」で新しい会話を開始(古いのはドロップダウンに残ります)

### 並列サブエージェント

> 「dispatch_agent を mode:"work" の parallel で2体使って、フロントとバックエンドを同時に進めて」

最大3体が同時に働きます。ヘッダ「エージェント」パネルで各自の状況が見え、
承認ダイアログには「🤖 サブエージェント #N」と出所が表示されます。
同じファイルを取り合うと2体目が拒否される安全装置つき。「停止」で全員止まります。

### 画像を見せる・見せて直させる(M14)

**あなたが見せる**: チャット欄に画像を**ドラッグ&ドロップ**または**ペースト**(Win+Shift+Sのスクショをそのまま貼れる)。
「このモックどおりに作って」「このエラー画面なにが悪い?」が可能です。スマホからは📷ボタンで写真を添付できます。

**AIが自分で見る**: `screenshot` ツールで、AIが作ったUIを**自分でスクショして見た目を確認・修正**します:

> 「カウンターアプリを作って、devサーバを起動して、スクショで自分の目で確認して、
>  ボタンが押しやすい見た目になるまで直して」

安全の仕組み:

- localhost のスクショは自動(読み取りのみ)。**外部URLは毎回あなたの承認**が必要
- 承認時に「セッション中許可(そのドメイン)」を選べます — 許可したドメインだけ自動化され、
  別ドメインは改めて確認。外部URLのスクショは全件監査ログに記録
- 画像はセッションあたり50MBまで保存(超えると古い画像から説明テキストに置換)

**fullPcのセッション中許可(M14-5・既定OFF)**: Settings → 操作範囲 のトグルをONにすると、
プロジェクト外への書き込み承認に「このフォルダは以後自動」ボタンが出ます(ツール×フォルダ単位。
bash等の実行系と保護領域は対象外・自動通過も全件監査記録)。放置作業が楽になる代わりに
チェックが減るので、信頼できる作業のときだけONに。

### AIの記憶(AMATERAS.md)

AIは作業中に見つけた知見(ビルド方法・規約・ハマりどころ)を、作業フォルダの
`AMATERAS.md` に**自分でメモします**(「## 学習メモ」節にタイムスタンプ付き)。
この記憶は毎回AIに渡されるので、次の依頼から同じ失敗を繰り返しにくくなります。

- あなたも Settings → プロジェクト記憶 から同じファイルを編集できます(共同のノート)
- 会話が長くなって古いやり取りが自動要約されるときも、消える範囲に大事な知見があれば
  AIが記憶へ退避してから畳みます
- 「◯◯を覚えておいて」と頼めばその場で追記します。肥大したら「記憶を整理して」でOK

## 自己進化 — AMA-terasの一番面白いところ

AIが「今の手持ちツールではできない」と判断すると(または、あなたが頼むと)、
**自分で新しいツールを作ります**。流れは全部見えます:

1. 進化パネルにジョブが現れる(別ブランチの隔離環境で作業するので、動いているアプリは無事)
2. 新ツールのコード+テストを自分で書き、検証ゲート(型チェック→テスト→実際に動かす)を通す
3. 全部合格すると**昇格承認ダイアログ**が出る — diffを見て、あなたが承認/却下
4. 承認するとアプリに組み込まれ、その場で使えるようになる。問題が起きたら自動で巻き戻し

頼み方の例: 「CSVをExcelに変換するツールが欲しい。request_capability を使って」

危険な要素(ネットワークアクセス・外部プロセス起動)が含まれる場合は、承認ダイアログに明示的な警告が出ます。

今どんなツールがあるか一覧で見たいときは Debug タブの「ツール一覧」で、
タグ(ファイル操作・Web操作・コマンド実行 等)による絞り込みや名前・説明での検索ができます。

**既にあるツールの修正**(M25-8): 「web_searchのこの挙動がおかしい、直して」のように頼むと、
新しく別のツールを作るのではなく既存のプラグインファイルを直接修正します。生成結果が
指定したツール名と食い違ったり、逆に新規のつもりが既存ツール名と衝突した場合は、
自動でフィードバックして直させるか、直らなければ失敗として扱われます(気づかず上書き・重複しない)。

### 本体の自己改変(M20: renderer / core スコープ)

ツール追加だけでなく、**アプリ本体の見た目(renderer)やロジック(core)も自己改変**できます。
進化パネルの手動起動でscopeを選ぶか、「UIのここを直して。進化機能で」のように頼みます。

- 隔離環境で生成された差分は7段のゲート(聖域チェック→危険検出→差分検査→型チェック→
  テスト→ビルド→実際にアプリを起動)を全部通らないと提案されません
- 提案は**赤枠の昇格ダイアログ**で出ます。「diffを確認し本体変更だと理解した」に
  チェックしないと承認ボタンが押せない二段確認です。**自律モード中でも絶対に自動承認されません**
- 承認すると再ビルド+健全性チェックのあとアプリが自動で再起動し、緑のバナーで完了を知らせます

**聖域(絶対に触れない場所)**: 承認の仕組み・進化システム自体・APIキー保存・CLAUDE.md などは
どのscopeでも変更できません。AIがそこに触れる差分を作った場合、承認ダイアログすら出ずに
「🛡 保護領域のため拒否」で即座に終わります(一覧: `docs/PROTECTED.md`)。

**失敗しても壊れない仕組み**:

- 再起動前の健全性チェックに失敗 → 自動で巻き戻し(アプリは動いたまま)
- 再起動後に2回連続クラッシュ → 3回目は**セーフモード**で起動(赤バナーに復旧コマンド表示・
  進化機能は一時停止)。バナーの「解除」ボタンで通常に戻せます
- 進化パネル下部の「昇格履歴」から「⏪ 1つ前へ戻す」でいつでも直前の進化を取り消せます

### プラグインの共有 — エクスポート / インポート(M27)

作ったツールを人に渡したり、共有されたツールを取り込めます(将来のコミュニティレジストリの土台)。

**エクスポート(書き出し)**: Debug タブのツール一覧 → 各ツールの「📦 エクスポート」→
保存先フォルダを選ぶだけ。「コード+テスト+manifest.json」の1フォルダとして書き出されます。
manifest には権限(ネットワーク/外部プロセス/ファイル使用)が**コードの自動解析で**記録されます。

**インポート(取り込み)**: 進化パネルの「📦 インポート」→ 共有されたフォルダ
(manifest.json を含む)を選ぶ。以下を**全部**通ってから初めて使えるようになります:

1. マニフェスト検査(形式・ライセンス・**宣言と実際のコードの権限が一致するか**。
   宣言していないネットワークアクセス等が見つかると、この時点で自動拒否)
2. 隔離環境での検証ゲート(型チェック→テスト→実際に1回動かす)— 自己進化と完全に同じ扉
3. 昇格承認ダイアログ(diff全文+危険API警告)— 最後は必ずあなたが決める

**キルスイッチ**: Settings → 品質タブの「プラグイン失効リストURL」を設定しておくと、
起動時にリストを確認し、危険と判明した共有プラグインを自動で無効化します(理由は
ツール一覧のエラー欄に表示。未設定なら何もしません)。

### 作る前に探す — レジストリ検索(M28)

Settings → **接続タブ** の「コミュニティレジストリURL」を設定しておくと、
エージェントが新しいツールを作ろうとしたとき、**生成の前にコミュニティの既存プラグインを
自動で探します**:

1. 候補が見つかると「コミュニティに既存の進化があります」という承認カードが出ます
   (名前・説明・作者・**検証済みバッジ**の有無。未検証の候補には追加の警告)
2. **許可** → ダウンロードして上記インポートと同じ検証ゲート(隔離環境で typecheck →
   テスト → 実行)+昇格承認を通ってから導入されます
3. **拒否** → 従来どおり自分で新しく作ります

レジストリが落ちていたり候補が無いときは、何も聞かずにそのまま生成へ進みます。
**無料モードでも検索・導入は使えます**(無効になるのは新規生成だけ)。
レジストリURLは既定で公式レジストリが設定済みです(接続タブで変更・無効化・既定に戻すが可能)。

### 仮導入と棚卸し — 放置作業でもツールが増やせる(M29)

自律モード(🔓)をONにするとき、**プラグイン導入の包括承認**を選べます:

| 範囲 | 意味 |
|---|---|
| 自動導入なし(既定) | 手持ちツール+新規生成のみ(生成の反映は従来どおり昇格承認待ち) |
| **検証済みのみ仮導入(推奨)** | レジストリの検証済み+危険権限なしのプラグインを、作業中に無人で「仮導入」 |
| 検証済み+生成も仮導入 | 上記に加え、自己生成したプラグインも無人で仮導入 |

仕組み(承認を消すのではなく、タイミングを作業の前後へ移す):

- 仮導入でも**検証3段(型チェック→テスト→実行)は必ず通ります**。危険権限
  (ネットワーク/外部プロセス)持ちは範囲に関わらず無人導入されません(個別の確認カードが
  出て、10分以内に応答がなければ諦めて従来の生成に進みます)
- 作業が終わると**棚卸しカード**が出ます: 「この作業で導入: ○○」→ 各項目を
  **残す**(確定)/**削除**(完全アンインストール=導入前の状態に巻き戻し)
- 未応答のまま閉じても大丈夫: 仮導入(ツール一覧に「仮」マーク)のまま残り、
  進化タブの「仮導入の棚卸し」に次回起動時も再提示されます

**カスタムテーマ(実験的・M27)**: Settings → 基本タブ下部の「カスタムテーマ(JSON)」に
配色JSONを貼ると見た目を変えられます。テーマは**コードではなくデータ**なので、
共有されたテーマを貼っても実行リスクはありません。

## PC全体モードと安全の仕組み

既定では、AIは**作業ディレクトリの中だけ**を触ります。
Settings → 操作範囲 を「fullPc」にすると、PC全体を対象にできます。ただし:

- **プロジェクト外の操作は、自動承認設定に関係なく毎回あなたに確認します**(「セッション中許可」も不可)
- 承認ダイアログに ⚠ プロジェクト外の警告バナーと、触ろうとしている実際のパスが表示されます
- Windowsシステム領域への書き込みや、アプリの認証情報(APIキー保存ファイル)へのアクセスは、
  確認すら出ずに**即拒否**されます
- プロジェクト外の操作はすべて監査ログ(audit.jsonl)に記録され、スマホの監査タブからも見られます

## MCPサーバーをつなぐ(外部ツール連携)

MCP(Model Context Protocol)対応の外部ツールサーバーをつなぐと、
そのツールがチャットからそのまま使えるようになります(GitHub操作、DB照会など)。

1. Settings → **MCPサーバー** 節で「名前 / コマンド / 引数」を入力して追加
   (例: 名前 `fs`、コマンド `npx`、引数 `-y @modelcontextprotocol/server-filesystem C:\dev`)
2. **信頼確認**が出ます — そのサーバーのツール実行を許可するか、あなたが決めます
   (許可するまで接続されません)
3. 接続されると「ツールn件」と表示され、AIが `mcp__fs__...` という名前で使えるようになります

安全の仕組み:

- MCPツールを使うときの承認ダイアログには「🔌 MCP: サーバー名」と出所が表示されます
- 危険度はサーバーの自己申告(annotations)から推定し、**申告が無いツールは最も厳しい
  exec 扱い**(=自動承認していない限り毎回確認)
- 接続に失敗してもアプリは普通に使えます(Settings に状態とエラーが出るだけ)
- 設定の実体は `%APPDATA%\amateras\mcp.json`(手編集も可)

## スマホから使う

1. PC・iPhone 両方に Tailscale を入れて同じアカウントでログイン(初回だけ)
2. Settings → リモートアクセス を有効化 → **表示されるQRコードをiPhoneのカメラで読むだけ**
   (トークン込みQRは有効化直後・トークン再生成直後にだけ表示されます)
3. 共有メニュー →「ホーム画面に追加」でアプリ化(トークンは保存されるので次回から開くだけ)

できること: チャット送信・進捗のライブ表示・**承認**(スマホで許可するとPC側も閉じる)・
自己進化の昇格承認・監査ログ閲覧。外出先から「あの作業やっといて」→ 承認だけスマホで、が可能です。

さらに(M23): **+新規ボタンで新規チャット**(実行中の作業は止まらない)・
実行中の**↩追加指示**・**設定タブ**(プロバイダ/モデル/自動承認/最大ターン/同時数の変更と
使用量の確認)。workspace・PC全体モード・編集後フック・APIキーはデスクトップ専用のまま
(スマホから変えられると危険なため)。

設定タブの「表示」でテーマ(ダーク/ライト)も選べます(この端末のブラウザだけに保存され、
サーバ設定ではないのでPCと揃える必要はありません)。

接続URLのトークンは Settings → トークン再生成 でいつでも無効化できます。

## 設定リファレンス(Settings ⚙)

| 設定 | 意味 | おすすめ |
|---|---|---|
| プロバイダ / モデル | Anthropic か OpenAI か、使うモデル | claude-fable-5(既定) |
| APIキー | 各プロバイダのキー(暗号化保存) | — |
| 作業ディレクトリ | AIの担当フォルダ | プロジェクトごとに切替 |
| 自動承認(safe/write/exec) | 危険度ごとに確認を省略 | 慣れたらON、放置作業は全ON |
| 最大ターン数 | 1回の依頼でAIが動ける回数(1〜200) | 普段30、長作業は100 |
| 編集後フック | ファイル編集のたびに走るコマンド | `npm run -s test --if-present` |
| 操作範囲 | project(既定)/ fullPc | 必要な時だけfullPc |
| リモートアクセス | スマホ用サーバのON/OFF・トークン・接続QR | 使う時だけONでも、常時ONでも |
| MCPサーバー | 外部ツールサーバーの追加・信頼・状態 | 使うものだけ追加し、都度信頼確認 |
| プロジェクト記憶 | AMATERAS.md の閲覧・編集(AIと共同管理) | AIの学習メモを時々眺める |
| fullPcセッション中許可 | プロジェクト外書き込みのフォルダ単位自動化(M14-5) | 既定OFF。放置作業時のみON |
| フォールバック | 残高・課金エラー時に別プロバイダで自動続行(M16) | 放置作業の前にON+切替先キー登録 |
| アニメーション | UIの表示効果のON/OFF(このPCにのみ保存) | 好みで。OSの「動きを減らす」設定が優先 |
| テーマ | ダーク/ライトの切替(このPCにのみ保存) | 好みで。スマホ側は設定タブに別トグルあり |

設定画面は **基本 / モデル運用 / 品質 / 接続 / 記憶** の5タブに分かれています(M26)。
テーマとアニメーションはヘッダにあります。claude-fable-5 選択中は「入力データは安全対策の
ため30日間保持(ZDR非適用)」の注意が表示されます。

そのほかの M26 の小さな追加:
- **進化ジョブのキャンセル**: 進化パネルの待機中/実行中ジョブに「✕ キャンセル」ボタン。
  待機中はキューから外れ、実行中は安全に中断されます(昇格承認待ち以降はダイアログの「却下」で)
- **会話の作業ディレクトリ移動**: 右ペイン上部の📁行の「移動」→フォルダ選択→確認で、
  この会話の作業場所を切り替えられます(実行中は不可。以降のツール実行が移動先を参照)

## 困ったとき

- **途中で止まった(max_turns_reached)** → 最大ターン数を増やして「続きをやって」
- **残高切れ・課金エラーで止まった** → APIクレジットを補充するか、Settings → フォールバック を
  ONにしておけば次からは自動で切替先が引き継ぎます
- **アプリが落ちた/閉じてしまった** → 再起動してセッションを復元。「続きをやって」で再開
- **AIが変なことをした** → Debugパネル → チェックポイントから復元。会話で「◯◯を元に戻して」でも可
- **承認ダイアログが出ない・進まない** → スマホ側で承認待ちになっていないか確認(どちらからでも応答可)
- **スマホがつながらない** → iPhoneのTailscaleがONか → Windowsファイアウォールで「プライベートネットワーク」を許可
- **進化ジョブが失敗した** → 進化パネルのログとゲート結果を確認。失敗してもアプリ本体は無傷(隔離環境で作業しているため)
- **進化の再起動後に赤い「セーフモード」バナーが出た** → 直前の進化が連続クラッシュを起こした
  合図です。バナーの復旧コマンド(`git reset --hard <直前コミット> && npm run build`)で確実に
  戻すか、「解除(要再起動)」で再挑戦できます。セーフモード中は進化機能だけが止まり、通常の
  チャット・ツールは使えます
- **進化を取り消したい** → 進化パネル下部「昇格履歴」の「⏪ 1つ前へ戻す」(直後のみ)。
  それ以前の進化は表示される手動手順(git revert)で戻します
- **2つ起動してしまった** → 2つ目は自動で終了し、既存ウィンドウが前に出ます
