## 既知の制約(仕様)

- セッションのロード/新規はエージェント実行中は不可(完了かキャンセル後に)
- セッションを復元しても workspace 設定は自動では切り替わらない(メタとして保存はされる。
  必要なら Settings で合わせる)
- work サブエージェントは plan / dispatch_agent / request_capability を使えない
  (計画は親が単独管理・ネスト1段・進化は親からのみ)
- 並列は最大3体。write 衝突テーブルは「最初に書いた子がそのパスを所有」方式
