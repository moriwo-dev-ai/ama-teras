# 体感確認ガイド(約15分・M12の残り項目)

自動テストでは確認済みの項目を「自分の目で見る」ためのコースです。上から順にやると15分程度。
チェックが済んだら `docs/M12-manual-test.md` の該当 `- [ ]` を `- [x]` にしてください。

## 準備(2分)

1. MyCodex を起動(すでに起動中ならそのまま)
2. 練習用の作業フォルダを作る: PowerShellで
   ```powershell
   mkdir C:\temp\mycodex-practice; cd C:\temp\mycodex-practice; git init; "node_modules/" | Out-File .gitignore -Encoding utf8
   ```
3. MyCodex の Settings → 作業ディレクトリ を `C:\temp\mycodex-practice` に変更
4. Settings → 自動承認 は全部OFFのままでOK(承認ダイアログを見るのも今回の目的)

## A. 計画ファイル(3分)

1. チャットに送信:
   > 「じゃんけんゲームのCLIを作って。まず計画を立てて、計画に沿って進めて」
2. **見るポイント**:
   - 最初の1〜2ターンで `MYCODEX_PLAN.md` が作業フォルダにできる
   - ヘッダの「計画」ボタンを押すとパネルが開き、`n/m 完了` のチェックボックス進捗が見える
   - 作業が進むたびに `- [ ]` が `- [x]` に変わっていく(ターン完了ごとに自動更新)
3. 完了までは見届けなくてOK。次のBでわざと中断します

## B. ツール実行中の強制終了→復元(3分)

1. Aの作業が動いている最中(ツール実行カードが出ているタイミングがベスト)に、
   タスクマネージャ(Ctrl+Shift+Esc)→ MyCodex → 「タスクの終了」で**強制終了**
2. MyCodex を再起動
3. チャット上部のセッションドロップダウンから、さっきのセッションを選んで復元
4. **見るポイント**:
   - エラーにならず履歴が全部戻る(実行途中だったツールは「アプリ再起動により中断」と表示される)
   - 「続きをやって」と送信 → 計画ファイルの続きから再開する(計画パネルの進捗も引き継がれている)
5. ついでに「新規セッション」ボタンを押す → 履歴がクリアされ、新しい会話として始まることも確認

## C. 並列サブエージェント+停止(3分)

1. チャットに送信:
   > 「dispatch_agent を mode:"work" の parallel で2体使って、`poem1.txt` と `poem2.txt` に別々の詩を書いて」
2. **見るポイント**:
   - ヘッダの「エージェント」パネルに子が2体現れ、それぞれの状態(実行中/完了)と現在のツールが流れる
   - 承認ダイアログに **「🤖 サブエージェント #1(#2)からの要求」** バナーが付く
3. 何回か承認して動きを見たら、**「停止」ボタン**を押す
4. **見るポイント**: 2体とも即座に止まる(エージェントパネルの状態がキャンセルに変わる)

## D. スマホからサブエージェント承認(2分)

1. もう一度Cと同じ指示を送る
2. 今度はPCの承認ダイアログを操作せず、**iPhoneのremote-ui**(ホーム画面に追加したやつ)を開く
3. **見るポイント**:
   - 承認タブにバッジが出て、サブエージェント出所バナー付きの承認カードが見える
   - スマホで「許可」→ **PC側のダイアログも自動で閉じて**作業が進む
4. そのまま完了まで流す(2ファイルができれば C の残り確認も兼ねて完了)

## E. 書き込み衝突と従来モード(2分)

1. 送信: 「dispatch_agent を mode:"work" の parallel で2体に、**同じファイル `shared.txt`** を書かせて」
2. **見るポイント**: 1体目は書けるが、2体目は「書き込み衝突」でエラーになる(両方成功したらバグ)
3. 送信: 「dispatch_agent でこのフォルダの構成を調査して」(mode指定なし)
4. **見るポイント**: 従来どおり読み取り専用で動き、書き込みの承認は一切出ない

## F. 進化ジョブ1本(様子見・5分/放置可)

M12-0(多重起動ガード)が進化のスモークゲートを壊していないことの最終確認です。

1. チャットに送信:
   > 「文字数・行数・単語数を数える `count_text` みたいなツールが欲しい。request_capability を使って」
2. 進化パネルでジョブの進行を眺める(worktree準備→生成→検証ゲート→昇格承認)
3. **見るポイント**: スモークゲートが合格すること(ここがM12-0の確認ポイント。A稼働中のheadless起動)
4. 昇格ダイアログが出たら diff を見て承認 → 新ツールがそのまま使えることを確認
