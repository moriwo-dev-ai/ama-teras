# M19 手動確認(品質レビュー・ゲート)

自動テスト(vitest 566件・typecheck 3構成)と実機実測(雑コード検出シナリオ+ベンチ②ON →
docs/autonomy-comparison.md)は 2026-07-06 に実施済み。以下は体感確認用。

## 設定

- [ ] Settings →「品質レビュー」をON → プリセット3種(しっかり/標準/軽め)・閾値・
      差し戻し上限・4軸チェックが出る
- [ ] OFF(既定)なら従来と完全に同じ(レビューは一切走らない)

## 発動と表示

- [ ] 「計画を立てて進めて」を含むタスクで、計画項目が [x] になった直後に
      チャットへ🧪レビューカードが出る(合格=緑 / 不合格=琥珀)
- [ ] カードを開くと4軸スコア(コード品質/見た目UX/要件達成/テストの質)と、
      file(場所): 問題 → 修正 形式の具体指摘が見える
- [ ] 計画を使わない短いタスク(ファイル1個作る等)では、完了時に1回だけレビューが走る
- [ ] 読み取りだけの会話(質問・調査)ではレビューは走らない
- [ ] CLIタスクなどUIが無い場合、「見た目/UX」軸は「—」(スキップ)になる
- [ ] Webアプリ+devサーバ起動中なら、レビュアーが screenshot でUIを確認することがある

## 差し戻しループ

- [ ] わざと雑に書かせる(「エラー処理・テストは書かなくていい」等)と、不合格カード→
      workerサブエージェントが修正(エージェントタブに表示)→再レビューで合格、が自動で回る
- [ ] 差し戻し上限(既定2回)でも閾値未満なら、赤い「残課題あり」カードが出る
  - 自律モードOFF: 承認ダイアログ「残課題を許容して先へ進みますか?」が出る
    (スマホの承認タブにも出る)。拒否すると「追加の修正指示を送って」と案内される
  - 自律モードON: 警告カードのみで自動続行
- [ ] `%APPDATA%\amateras\audit.jsonl` に `"tool":"review-gate"` の行が
      ラウンドごとに記録されている(milestone/round/avg/pass/findings)

## 併存・非波及

- [ ] M18のモデル自動切替ONなら、レビューはplanner帯・修正はworker帯で走る
      ([usage]ログに session=review-N が出る)。M18 OFFなら本体モデルで実行
- [ ] 進化ジョブ(request_capability)には品質レビューは適用されない(既存4ゲートのみ)
- [ ] レビュー中に「停止」を押すと全体がすぐキャンセルされる
